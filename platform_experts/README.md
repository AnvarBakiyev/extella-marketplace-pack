# Платформенные эксперты MCP — версионный контроль

Эти эксперты жили ТОЛЬКО в базе Extella (созданы на платформе 08–09.07, исходников
не было ни в одном репозитории). На них опираются `toolbar.js` (`mcp_list` — состояние
подключений) и визард (`wz_capability_install`, обёртки инструментов). Одна неудачная
перезапись — и код было бы не с чем сравнить и нечем откатить.

**Здесь — источник правды.** Правишь эксперта → правь файл здесь → заливай через REST
`/api/expert/save {name, description, code, global:true}` → коммить.

Лежит НЕ в `experts/`: содержимое `experts/` установщик пака заливает клиентам, и
копия затирала бы правки, сделанные на платформе другими.

## Состав

| Эксперт | Что делает |
|---|---|
| `mcp_call.py` | Вызов инструмента MCP-сервера; `tool:"__list__"` — перечисление инструментов |
| `mcp_connect.py` | Поднимает пакет (npx/uvx), рукопожатие, запись в платформенный каталог данных Extella |
| `mcp_list.py` | Читает аллоулист устройства → `{servers:[{key,pkg,title,count,tools}]}` (это читает тулбар) |

## Журнал правок

- **09.07** — созданы на платформе (Toolbar-сессия): подключение MCP с устройства,
  тулбар перестал врать хардкодом `['fetch','time','git','filesystem']`.
- **19.07 (Wizard-чат)** — в `mcp_call.__list__` добавлено поле `schema` (урезанная
  `inputSchema`: поле → тип + пояснение ≤100 симв., `required`, максимум 20 полей).
  Нужно, чтобы генератор обёрток не угадывал имена аргументов.
- **19.07 (Toolbar-чат)** — забраны под версионный контроль сюда. Проверено, что
  правка обратно совместима: `__list__` по-прежнему отдаёт `name`/`desc`
  (868 байт на сервере `time`), `mcp_list` — вообще без схем (~1.7 КБ, ключи
  `key/pkg/title/count/tools`), а тулбар читает только его → утяжеления UI нет.

## Ловушки

- `/api/expert/get {name, global:true}` отдаёт `expert_code` целиком — так и снимался
  этот бэкап. (Канонная заметка «expert/get не отдаёт инструкции» — про АГЕНТОВ.)
- Сохранять только с `global:true` — иначе эксперт станет личным и у других пропадёт.
- MCP-инструменты `save_expert`/`get_kv` падают («Failed to resolve dependency 'user_id'») —
  работать через REST.
