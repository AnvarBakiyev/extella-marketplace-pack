# Rules — Extella Marketplace (способности + канон)

---

## rule: services_protocol
У агента есть установленные инструменты-эксперты `svc_*` для живых данных (курсы валют, крипта, погода, перевод, справки, новости, экономика стран, праздники, GitHub, книги, гео, QR). Когда задача требует таких данных — ОБЯЗАТЕЛЬНО вызвать соответствующий `svc_*` через run_expert и ответить строго по его фактическому результату; не выдумывать числа и не подменять веб-поиском. Инструменты можно комбинировать в цепочку (например, взять курс через `svc_currency`, затем посчитать сумму). Полное описание — concept «Сервисы». Эксперты `svc_*` доступны агенту, только если сохранены в его скоупе (save_expert с X-Agent-Id = id агента).

---

## rule: one_tool_per_turn
За один ход агент вызывает только ОДИН инструмент, дожидается результата и цитирует ФАКТИЧЕСКИЙ результат вызова. Запрещено писать «готово / отправлено / сделал», если результат вызова не получен и не показан (защита от фабрикации у Qwen 3.7). Длинные задачи дробить на явные шаги.

---

## rule: drafts_and_honesty
Любое внешнее действие (отправка письма, публикация, запись во внешнюю систему) агент готовит как ЧЕРНОВИК и просит подтверждение — финальное действие выполняет человек. Не выдумывать факты и цифры: если данных нет — честно сказать и предложить, что нужно.

---

## rule: knowledge_on_device
Базы знаний `kp_*` (RAG) и CLI-инструменты `cap_*` выполняются НА УСТРОЙСТВЕ пользователя (локальный Ollama, файлы) — их запускают через мост тулбара/устройство, а не через облачный run_expert. Ответы по базе знаний — строго по её фрагментам, с указанием источника.
